## Описание

Приложение на C++ с графическим интерфейсом Qt для приёма данных от модели (PID-регулятор с шумами и выбросами), фильтрации сигнала с помощью КИХ и БИХ фильтров в отдельных потоках и отображения исходных и обработанных данных на графике в реальном времени.

Взаимодействие с моделью осуществляется по протоколу UDP. Приложение отправляет целевое значение (float, 4 байта) и принимает пакеты, содержащие временную метку (uint32_t) и текущее значение (float). Реализованы два типа фильтров, параметры которых могут настраиваться пользователем через GUI. Визуализация отображает последние N отсчётов (N настраивается).

## Функциональные возможности

- Приём потока данных от модели по UDP.
- Отправка целевого значения в модель по UDP.
- Настройка сетевых параметров (адрес, порт) для приёма и отправки.
- Реализация КИХ-фильтра (например, скользящее среднее) и БИХ-фильтра (экспоненциальное сглаживание) в отдельных потоках.
- Возможность настройки параметров фильтров через GUI (размер окна, коэффициент сглаживания и т.д.).
- Отображение на графике исходного сигнала и сигналов после каждого фильтра.
- Настройка количества отображаемых отсчётов (50–1000).
- При необходимости: расчёт и визуализация спектра (БПФ), автоматическое циклическое задание целевого значения, анализ вычислительной сложности.

## Требования для сборки

### Обязательные компоненты

- Операционная система: Windows 10/11, Ubuntu 18.04 или новее.
- Компилятор с поддержкой C++17 (GCC 7+, Clang 6+, MSVC 2019+).
- CMake версии 3.16 или выше.
- Qt 5.12 или новее (модули: Core, GUI, Widgets). Рекомендуется Qt 6.x.
- Python 3 (только для запуска модели, сама модель не требует дополнительных пакетов).

### Дополнительные зависимости (для БПФ)

- FFTW3 (если реализован спектральный анализ).

## Сборка и запуск
```bash
1. Клонирование репозитория

git clone https://github.com/FedorKulikov/Signal.git
cd Signal
2. Настройка CMake
Создайте директорию для сборки и выполните конфигурацию:

bash
mkdir build && cd build
cmake ..
Если Qt не найден автоматически, укажите путь вручную:

bash
cmake .. -DCMAKE_PREFIX_PATH=/path/to/Qt
3. Сборка
bash
cmake --build . --config Release
На Linux можно использовать make:

bash
make -j$(nproc)
4. Запуск модели
Перед запуском приложения необходимо запустить модель (предоставленный файл model.py):

bash
python model.py
В окне модели при необходимости можно изменить параметры PID, шума, выбросов и сетевые настройки.
Значения по умолчанию:

Приём команд (UDP): 127.0.0.1:50005

Отправка данных (UDP): 127.0.0.1:50006, период 20 мс

5. Запуск приложения
Выполните в директории build:

Windows: Release/Signal.exe

Linux: ./Signal

В приложении установите сетевые параметры, соответствующие настройкам модели:

Приём данных – адрес и порт, куда модель отправляет данные (по умолчанию 127.0.0.1:50006).

Отправка цели – адрес и порт, где модель принимает команды (по умолчанию 127.0.0.1:50005).

Нажмите кнопку «Применить» или аналогичную для активации сетевых подключений.

Использование
В поле Target Value введите числовое значение и нажмите Send. Модель начнёт движение к этой цели.

На графике отображаются три линии:

Raw – исходные данные от модели (синий).

FIR – результат работы КИХ-фильтра (красный).

IIR – результат работы БИХ-фильтра (зелёный).

В панелях настройки фильтров измените параметры (например, размер окна КИХ или коэффициент сглаживания БИХ) и наблюдайте изменение сигналов.

Поле Buffer size позволяет задать количество отображаемых отсчётов (50–1000).

При необходимости измените сетевые адреса и порты и нажмите Reconnect (или аналогичную кнопку).

Архитектура приложения
Сетевой уровень: реализован на стандартных сокетах POSIX (Windows: Winsock2). Приём данных выполняется в отдельном потоке.

Многопоточность: каждый фильтр (КИХ и БИХ) работает в своём потоке (std::thread или QThread). Это позволяет не блокировать основной поток GUI.

Графический интерфейс: Qt Widgets, обновление графика происходит по таймеру с заданной частотой.

Фильтры:

КИХ: реализован как фильтр скользящего среднего (окно усреднения). Размер окна настраивается.

БИХ: реализован как экспоненциальное скользящее среднее (EWMA) с коэффициентом α.

Обработка данных: все полученные данные сохраняются в кольцевых буферах фиксированного размера для отображения и фильтрации.
