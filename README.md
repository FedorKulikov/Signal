# Signal Filtering Application

**Приложение для приёма, фильтрации и визуализации сигналов в реальном времени**

Приложение разработано для работы совместно с моделью динамической системы (`model.py`).

---

## Описание проекта

Данное приложение принимает поток данных от Python-модели (PID-регулятор + шум + выбросы), применяет к ним цифровые фильтры в отдельных потоках и отображает результаты на графике в реальном времени.

### Основные возможности

- Отправка целевого значения в модель по UDP
- Приём данных от модели по UDP
- Реализация КИХ и БИХ фильтров
- Каждый фильтр работает в отдельном потоке (`QThread`)
- График реального времени с несколькими сигналами (разные цвета)
- Настройка параметров фильтров и сетевых настроек через GUI

---

## Требования

### Системные требования
- **Операционная система**: Windows 10/11 или Linux (Ubuntu 20.04+)
- **C++**: стандарт **C++17**
- **Qt**: версия **6.2** и выше (рекомендуется **Qt 6.5+**)
- **CMake**: версия **3.16** и выше

### Компоненты Qt
- `Qt6::Core`
- `Qt6::Gui`
- `Qt6::Widgets`
- `Qt6::Network`
- `Qt6::Charts` (или QCustomPlot)

## Сборка проекта

### 1. Клонирование репозитория
```bash
git clone https://github.com/FedorKulikov/Signal.git
cd Signal
```

### 2. Сборка

**Windows**
```bash
mkdir build && cd build

cmake .. -G "Visual Studio 17 2022" ^
  -DCMAKE_PREFIX_PATH="C:/Qt/6.6.0/msvc2019_64" ^
  -DCMAKE_BUILD_TYPE=Release

cmake --build . --config Release
```
> **Важно:** Измените путь `CMAKE_PREFIX_PATH` на актуальный путь к вашей версии Qt.

**Linux (Ubuntu / Debian)**
```bash
sudo apt update
sudo apt install build-essential cmake ninja-build \
                 qt6-base-dev qt6-base-dev-tools \
                 qt6-charts-dev libgl1-mesa-dev

mkdir build && cd build
cmake .. -DCMAKE_BUILD_TYPE=Release -G Ninja
cmake --build .
```
